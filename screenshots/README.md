# Streamlit Dashboard Screenshots

This directory is reserved for capturing and displaying visual assets of the **FloodGuard** Streamlit application on GitHub. 

To complete your portfolio representation, run the app locally (or in Docker) and save screenshots of the dashboard interface into this folder using the exact filenames listed below.

---

## Filenames and Capture Instructions

### 1. `app_homepage.png`
*   **What to capture:** The initial loaded state of the dashboard, showing the main header, page layout, and side panel controls before modifying rainfall sliders.
*   **Visual location in README:** Placed under the **Project Overview** section.

### 2. `dashboard.png`
*   **What to capture:** The full-width preview of the dashboard screen showing the stats indicators, map canvas, and sidebar options in one view.
*   **Visual location in README:** Placed under the **Dashboard Preview** section.

### 3. `flood_risk_visualization.png`
*   **What to capture:** A zoomed-in view of the interactive **PyDeck 3D map** rendering columns with HSL colors (Green, Orange, Red) representing Low, Medium, and High risk classes.
*   **Visual location in README:** Placed under the **Interactive Map & Visualization** section.

### 4. `prediction_output.png`
*   **What to capture:** The risk classification snapshot metrics summary grid (Low, Medium, High Risk Zones count) showing the real-time simulation output.
*   **Visual location in README:** Placed under the **Model Predictions** section.

---

## File Format Tips
*   **Resolution:** Capture at standard 1920x1080 resolution or crop to application boundaries.
*   **Format:** Save files strictly as `.png`.
*   **Size:** Keep file sizes below 1MB for fast loading on GitHub.
